### Ограничение вывода (limit pipe)

Если требуется обработать лишь часть выбранных логов, можно использовать конструкцию `| limit N` [пайп](https://docs.victoriametrics.com/victorialogs/logsql/#pipes), где `N` может принимать любое [поддерживаемое целочисленное значение](https://docs.victoriametrics.com/victorialogs/logsql/#numeric-values).

Например, следующий запрос вернёт не более 100 логов за последние 5 минут:

```logsql
_time:5m | limit 100
```

Для удобства вместо `limit` можно использовать ключевое слово `head`. Например, `_time:5m | head 100` эквивалентно `_time:5m | limit 100`.

Значение `N` в конструкции `head N` можно опустить — в этом случае будет возвращено не более 10 подходящих логов:

```logsql
error | head
```

По умолчанию строки выбираются в произвольном порядке — это сделано для повышения производительности, поэтому при каждом выполнении запроса выше может возвращаться разный набор логов.

Чтобы гарантировать одинаковый порядок логов перед применением `limit ...`, можно использовать [пайп `sort`](https://docs.victoriametrics.com/victorialogs/logsql/#sort-pipe).
