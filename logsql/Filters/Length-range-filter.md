### Фильтр по диапазону длины

Если требуется отфильтровать сообщения журнала по их длине, можно использовать фильтр `len_range()`.

Например, следующий запрос на языке LogsQL отбирает `сообщения журнала`, длина которых находится в диапазоне `[5, 10]` символов:

```logsql
len_range(5, 10)
```

Этот запрос подойдёт для следующих сообщений журнала, поскольку их длина попадает в указанный диапазон:

- `foobar`
- `foo bar`

Этот запрос **не** подойдёт для следующих сообщений:

- `foo` — слишком короткое;
- `foo bar baz abc` — слишком длинное.

Можно использовать `inf` в качестве верхней границы. Например, следующий запрос отбирает `сообщения журнала` длиной не менее 5 символов:

```logsql
len_range(5, inf)
```

Границы диапазона можно задавать в следующих формах:

- **Целочисленная форма с разделителями `_` для лучшей читаемости.** Например: `len_range(1_000, 2_345_678)`.

По умолчанию фильтр `len_range()` применяется к полю ``_msg``.

Чтобы применить фильтр к нужному полю, укажите `имя поля` перед `len_range()`. Например, следующий запрос отбирает записи журнала, у которых длина поля `foo` находится в диапазоне `[10, 20]` символов:

```logsql
foo:len_range(10, 20)
```
